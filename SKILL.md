# Weekly Report Skill — Gnatus/Branevo

Skill operacional para criação, manutenção e evolução do **Report Semanal Gnatus/Branevo**. Documenta padrões oficiais de estrutura visual de slides 16:9, modo apresentação, animações canônicas, fluxo de dados semanal, e templates de novas semanas e novos slides.

## Princípio principal

Não improvisar estrutura visual, animação ou fluxo de dados. O Report Semanal é um artefato **executivo e cinematográfico**, com padrões fechados.

## Quando usar

Sempre que for pedir para uma IA:

- criar uma nova semana do report;
- adicionar slide recorrente;
- ajustar modo apresentação;
- mexer em `MeetingHero`, `BuTitleSlide`, `BuOpener`, `ReportTemplate` ou slides de BU;
- consumir dados de `/api/semanal/*`;
- padronizar animações e estrutura visual.

## Referências

Antes de fazer qualquer alteração, leia:

- `references/visual-system.md` — direção visual, layout 16:9, tipografia, paleta, motion, narrativa cinematográfica e padrão executivo.

## Estrutura visual resumida (ver `references/visual-system.md` para detalhes)

- **Layout**: 16:9 widescreen, grid consistente, grandes áreas de respiro, hierarquia clara.
- **Tipografia**: sans-serif premium (Inter, Geist, Satoshi, SF Pro, General Sans), monoespaçada opcional para métricas.
- **Paleta**: fundos escuros ou neutros, branco/off-white para contraste, cor de destaque só para narrativa.
- **Motion**: fade, translate suave, blur leve, stagger discreto. **Nunca** bounce, zoom agressivo ou rotação.
- **Narrativa por slide**: (1) o que aconteceu, (2) por que importa, (3) qual decisão influencia.

## Evitar

- excesso de cards/texto/bullets;
- visual varejista ou template genérico de Canva;
- múltiplas cores competindo, gradientes exagerados, saturação alta;
- animações caricatas.
